What is CogNexus

CogNexus is a personal knowledge management platform with AI Agent capabilities. It uses a three-layer architecture to store fragmented knowledge, discover hidden connections, and generate higher-level abstractions. You can also create Character Agents to chat with AI personas grounded in your knowledge.

💡

CogNexus is currently in experimental stage. Features and APIs may change at any time.

Three-Layer Knowledge Architecture

CogNexus organizes knowledge in three layers, each building on the previous:

📝 Fact Layer

Raw knowledge fragments — notes, quotes, observations, data points. Stored in SQLite (metadata) + Qdrant (bilingual semantic vectors via BGE-M3). Supports fast keyword and semantic search across all your knowledge.

🔗 Relation Layer

Connections between facts discovered via Neo4j knowledge graph. AI automatically identifies relationships — cause-effect, similarity, contradiction, temporal sequence. Browse the graph to see how your ideas connect.

🧠 Abstraction Layer

Higher-level insights auto-generated from patterns in your knowledge. Use the /abstract command to trigger abstraction generation. View them in Tree view to see your knowledge hierarchy.

Agent Types

CogNexus supports two types of Agents:

👤 Human

Your personal knowledge base agent. Created automatically on registration. Each Human Agent has its own namespace, knowledge store, and LLM configuration. It's your default identity on the platform.

🎭 Character

AI role-playing agent with a configured persona. Can be historical figures (Laozi, Feynman), fictional characters, or domain experts. Characters are built on curated knowledge bases and respond from their unique perspective.

Each Agent has a unique namespace that isolates its knowledge store. Your Human Agent's namespace defaults to your username (lowercase).

Knowledge Management

You interact with your knowledge base through chat and slash commands:

Command Description
/research [topic] AI searches the web and stores findings as facts
/today Review today's knowledge entries
/abstract Generate abstractions from existing knowledge
/search [query] Search across all knowledge by keyword or semantics

You can also store knowledge by simply chatting — the system automatically extracts facts from conversation and stores them in the appropriate layer.

Character Agents

To create a Character Agent:

  1. Go to Dashboard and click "Create Agent"
  2. Choose type "Character" and set a namespace
  3. Configure the persona — name, background, personality, speaking style
  4. Add knowledge to the character's namespace (facts, writings, research)
  5. Chat with the character — it responds based on persona + knowledge
💡

Characters can auto-research based on their identity. Use /research in a character's chat to have them gather knowledge aligned with their persona.

Visualization

CogNexus provides multiple ways to visualize your knowledge:

  • Graph View — Force-directed graph showing facts as nodes and relations as edges. Interactive zoom, pan, and click to explore connections.
  • Tree View — Hierarchical view of the Abstraction Layer. See how high-level concepts break down into supporting facts.
  • Timeline — Chronological view of knowledge entries. Track how your understanding evolved over time.
  • Chat — Conversational interface to query, add, and explore knowledge interactively.

LLM Configuration

CogNexus supports multiple LLM providers:

Provider Models
Anthropic Claude Sonnet, Claude Opus, Claude Haiku
OpenAI GPT-4o, GPT-4o-mini
DeepSeek DeepSeek Chat, DeepSeek Reasoner
Doubao Doubao Pro, Doubao Lite

Set a global default in Settings → LLM Config. Each Agent can override with its own provider and API key via the Agent Config tab.

Token Mechanism

Tokens are credentials for accessing Agents. Different token types have different permissions:

Type Permission Description
full Full Access Browse knowledge + Q&A + Chat + All views
qa_public Public Q&A Limited Q&A sessions with usage quota
browse_public Browse Only View public knowledge content only, no interaction

A default browse_public Token is created automatically when you register. You can create additional Tokens in the Agent detail page → Token management.
